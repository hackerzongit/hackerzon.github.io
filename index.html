Sprint Structure

12 weeks.
4 project sprints.

Shown here for the Web Application Security domain — each sprint builds on the last, culminating in a full assessment and professional vulnerability report.

S1
WEEKS 1 – 3
Recon & Web Fundamentals
HTTP internals, web architecture, passive/active recon, Burp Suite setup. You'll learn to read web traffic, enumerate attack surfaces, and perform your first structured assessment.
HTTP Deep Dive Burp Suite OWASP Top 10 Google Dorking Gobuster
S2
WEEKS 4 – 6
Injection Attacks
Deep-dive into SQL injection (manual & sqlmap), all three XSS types, SSTI, and command injection. PortSwigger labs throughout. By week 6 you can exploit a vulnerable app end-to-end.
SQLi Manual sqlmap XSS Masterclass SSTI Command Injection
S3
WEEKS 7 – 9
Authentication & Session
Authentication bypass, session hijacking, JWT attacks, OAuth 2.0 misconfigurations, and IDOR vulnerabilities. You'll complete a full auth audit on a multi-user application.
Auth Bypass Session Hijack JWT Attacks OAuth IDOR
S4
WEEKS 10 – 12
Full Assessment & Reporting
End-to-end web application penetration test. CVSS scoring, CVE identification, MITRE ATT&CK mapping, and writing a professional vulnerability report — the same format used by real consultancies.
Full VAPT CVSS Scoring MITRE ATT&CK Report Writing Remediation
